Fast Motion Planning

Fast and robust motion planning is an essential requirement for efficient robot deployments such as a robot arm or mobile robot. During my time at RVBUST, I developed a method to embed deep neural networks into the RRT algorithm. The result suggests that our algorithm can save up to 70% of the time compared to the state-of-the-art motion planning algorithm.

Data Generation

(This is a 2d C-space example in order to demonstrate the whole process)

After setting up the simulated scene, which is the same as the real-world environment, the data set is generated by Fast Collision Library (FCL). To generate a data set that can better represent the boundary between the in-collision subset and collision-free subset, we designed a sampling strategy by focusing on the boundary between the in-collision region and collision-free region.

RRT Embeded with Neural Network

Instead of directly using collision check libraries like FCL or Bullet in the RRT algorithm, we use the Neural Network as a filter to decide whether an accurate but expensive kinematic collision check is necessary. If the Neural Network is very confident that there will be or will not be a collision, then we trust its result and proceed to the next configuration point. If the prediction is ambiguous, an accurate collision check will be applied.

After the first planning, we will go through the generated path to see if there is any invalid segment. Once the segment is repaired, we will guarantee to get a safe path.
